Fonts! Fonts! Fonts!

The easiest, fastest thing you can do to show your readers you've put some thought into the design of your page is to use a custom font. Almost everyone associates the default browser fonts with sloppy, amateur design. In this exercise we'll learn about the font-family css property and how to use it. Important!!! The BEST way to incorporate fonts into web documents is to use web fonts, which are discussed in the last section of this document. I discuss generic font-family names and local fonts so that you will be aware of them, and so you'll see why web fonts are so powerful. However, these should be used as fallback fonts for web fonts, not as the primary source of fonts in your design.

The font-family property and generic families

We set the typeface (what a word-processor would call the font) of our text using the font-family property. Take a look at the documentation for font-family on MDN. In short, font-family lets us specify lists of fonts in order of priority. The browser will try each font in the list, in order, from the first listed to the last, until it finds one that is either installed on the user's computer (local font) or available from a server on the web (web font). It will then apply this font to the elements selected by our CSS rule.

Take a look at the lists of fonts provided in the examples on the MDN documentation page. Notice that the last font listed is always one of the generic family names, listed below:

  • serif
  • sans-serif
  • monospace
  • cursive
  • fantasy

Serif and sans-serif you should recognize from our reading. (NOTE: Oldstyle, slab-serif, and modern fonts are all serif fonts.) "Cursive" is basically what the book call "script," and "fantasy" is effectively "decorative." These generic family names tell the browser to pick something from the broad family of fonts if nothing else is available for use.

Local Fonts

In addition to the generic family names, we can also specify a specific font we want the browser to use by name. However, the user must have this font installed on his or her computer for the font to display correctly. In other words, if we choose a font we have installed on our own computer for our design, but the user who visits our web page does not have this font installed on his or her computer, the font will not display. This can cause real headaches, which is why we rely on web fonts, discussed below.

When we do use local fonts, we rely on web-safe fonts whenever possible. Web-safe fonts are fonts that are widely installed, so we can expect most visitors to our site to have them available. This page is a good resource for web-safe fonts. Click on any of the named "font stacks" to see css that incorporates a font-family rule that attempts to approximate the named font on most computers. The percentage numbers below each named stack show what percentage of macintosh and windows computers have a font from this stack installed.

Using Google Fonts As a Web Font Provider

To avoid some of the headaches of font-stack design and to ensure that the font we select as designers is the font our users see, we employ web fonts. Web fonts are provided by a font service, which makes fonts available on server. Since the fonts are availble on a server, they do not need to be installed locally on the user's computer (however, if something causes the font service to go down, the fonts will become unavailble and stop displaying). There are many font services out there, including Adobe's TypeKit service, which many designers like. However, most services charge by the font for use, and can get a bit expensive. For that reason, we will use Google's font service, which is free of cost, for this class.

For each Google font you want to use, you must include 2 lines of code in the head section of your page. The first line is a link element that includes the font information from Google's servers. The second is a font-family rule that tells the browser where you want to apply the font on the page. Try this out now. Include the font "Alegreya" and use it as the default font for all elements by adding this line before our style section:

<link href='http://fonts.googleapis.com/css?family=Alegreya' rel='stylesheet' type='text/css'>

And then add the following rule to our style section:

body {
font-family: 'Alegreya', serif;
}

Browse through the available fonts on Google's font service and find one you like. Then set this font, instead of Alegreya as the font for the body of this page. To find the information you need to accomplish this: click the "quick use button" for the font you want to use, which is the second button in the row of three small grey buttons to the left of the blue "Add to collection" button. (Ask if you need me to point it out.) On the next page, the "add this code to your website" section gives the link you need to include the font information and the "integrate the fonts into your CSS" section gives the font-family property for your style section.
